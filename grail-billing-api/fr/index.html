Ce que démontre cette API

La référence ci-dessous est générée depuis la même source OpenAPI que Swagger, puis enrichie pour une lecture portfolio.

Cycle de vie facture

Brouillons, émission officielle, informations verrouillées, paiements et statut consultable.

Contexts bornés

Auth, Billing, Companies, Customers, Catalog et Reference Data restent séparés.

Démo opérationnelle

Readiness, métriques admin-only, healthchecks Docker et reset déterministe.

Workflow développeur

Swagger reste concis pour les tests locaux ; cet export est optimisé pour la publication en lecture seule.

Workflow

Parcours de lecture conseillé

  1. S'authentifier avec un compte demo ou admin.
  2. Créer les données support : company, customer, tax rate et product.
  3. Créer une facture brouillon depuis des données de requête validées.
  4. L'émettre pour verrouiller les informations et assigner une référence.
  5. Enregistrer les paiements et lire la projection facture.
