Sécurité & données

Limiter les données sensibles et garder le contrôle côté serveur.

Le prototype évite de conserver les CV sources et le texte brut extrait. Les clés API et détails techniques restent côté serveur.

CV non conservés

Le PDF est stocké temporairement, le texte est extrait, puis le fichier est supprimé avant l’appel OpenAI.

Texte brut non persisté

Le texte extrait est conservé uniquement en mémoire durant l’analyse.

Clé OpenAI serveur

La clé est configurée dans l’admin WordPress et n’est jamais exposée au frontend Nuxt.

Accès contrôlés

Les utilisateurs doivent être actifs pour Grail HR. Les niveaux V1 sont recruteur et manager.

Prototype / demo

Avant un usage production, il faudrait compléter par un audit sécurité, une politique RGPD formalisée, des tests, une revue des logs et une stratégie de sauvegarde.

Prototype fonctionnel développé comme démonstration produit et technique. Données fictives uniquement.
