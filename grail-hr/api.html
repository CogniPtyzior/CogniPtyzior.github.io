API REST

Une API sécurisée pour le frontend Nuxt.

Le frontend consomme l’API du plugin avec un bearer token. Les permissions sont vérifiées côté serveur avant chaque action protégée.

EndpointUsage
POST /auth/loginConnexion avec identifiants WordPress et création d’un bearer token.
GET /meRetourne l’utilisateur courant et ses permissions Grail HR.
GET /profilesListe paginée, filtrée et triée côté serveur.
POST /profilesCréation manuelle d’un profil éditable.
POST /profiles/from-cvCréation depuis CV PDF et analyse IA dans le même flux.
GET /profiles/{id}Détail complet d’une fiche profil.
PUT /profiles/{id}Mise à jour des données relues.
POST /profiles/{id}/validateValidation humaine du profil.
POST /profiles/{id}/archiveArchivage du profil.
GET /analysis/statusStatut d’analyse et suivi côté frontend.
GET /exports/...Export JSON ou CSV des profils autorisés.
Authentification

Le token est opaque côté client. Seul son hash est stocké côté WordPress.

Prototype fonctionnel développé comme démonstration produit et technique. Données fictives uniquement.
