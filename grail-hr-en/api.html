REST API

A secured API for the Nuxt frontend.

The frontend consumes the plugin API with a bearer token. Permissions are checked server-side before every protected action.

EndpointPurpose
POST /auth/loginSign in with WordPress credentials and create a bearer token.
GET /meReturn the current user and Grail HR permissions.
GET /profilesServer-side paginated, filtered and sorted list.
POST /profilesCreate a manual editable profile.
POST /profiles/from-cvCreate a profile from a PDF resume and run AI analysis in the same flow.
GET /profiles/{id}Full profile detail.
PUT /profiles/{id}Update reviewed data.
POST /profiles/{id}/validateHuman validation of the profile.
POST /profiles/{id}/archiveArchive the profile.
GET /analysis/statusAnalysis status for frontend tracking.
GET /exports/...JSON or CSV export of authorized profiles.
Authentication

The token is opaque on the client side. Only its hash is stored in WordPress.
