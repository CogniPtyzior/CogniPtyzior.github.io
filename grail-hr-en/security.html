Security & data

Reduce sensitive data exposure and keep control server-side.

The prototype avoids storing source resumes and extracted raw text. API keys and technical details remain on the server.

Resumes are not kept

The PDF is stored temporarily, text is extracted, then the file is deleted before the OpenAI call.

Raw text is not persisted

Extracted text is held in memory only during analysis.

Server-side OpenAI key

The key is configured in WordPress admin and is never exposed to the Nuxt frontend.

Controlled access

Users must be active for Grail HR. V1 access levels are recruiter and manager.

Prototype / demo

Before production use, the project would require a security audit, formal data-protection policy, testing, log review and backup strategy.

Functional prototype built as a product and technical demo. Fictional data only.
