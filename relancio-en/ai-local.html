Local AI

Understand Ollama, local models, timeout and regeneration.

Understanding local AI

Relancio uses Ollama to query a model installed on the user’s computer. The typical local address is http://localhost:11434.

The model receives useful information from the email and local business rules in order to produce a proposed reply. This proposal remains editable and must be validated by the user.

Installing the AI assistant

If the local AI assistant is not installed yet, Relancio offers automatic installation. If this installation does not work on the computer, you can download Ollama from the official website, then return to Relancio and refresh the models.

Timeout and fallback

The timeout defines how long Relancio waits for the model response. If Ollama does not respond or exceeds the timeout, Relancio sends nothing automatically: you can edit manually, save or launch a new generation.

Regenerate

The Regenerate button asks the local AI for a new proposal for the selected email. It is useful after changing context or if the first reply is not satisfactory.

Validation responsibility

A generative AI can produce an inaccurate, incomplete or poorly adapted reply. Always review the content, subject and recipient before sending.
